    - 사례: Claude Code가 JS 오케스트레이션 스크립트를 자동 작성해 최대 1,000개 subagent를 병렬 실행. Jarred Sumner는 이를 사용해 Bun 코드베이스 75만 줄(Zig→Rust 포팅)을 11일 만에 완료(기존 테스트 99.8% 통과). subagent 결과가 부모 컨텍스트로 쌓이지 않아 대규모 작업에서 품질 열화 방지.
    - 접목: 이상치 탐지 배치에서 게임 테이블별 서브에이전트 할당 — "각 테이블 담당 에이전트가 독립 실행 후 결과만 취합"하는 패턴으로 Snowflake 쿼리 병렬화 및 브리핑 속도 단축.

    - 접목: 게임 유저 매크로/어뷰저 탐지에 그대로 적용 가능. 레이블 없는 상태에서 행동 패턴 이상치(속도·반복성·시간대 패턴)를 앙상블로 검출, 현재 규칙 기반 탐지보다 미탐 감소 기대.

- [Claude Code Multi-Agent tmux Setup](https://medium.com/medialesson/claude-code-multi-agent-tmux-setup-7361b71ff5c4) · Medium

    - 접목: 헤드리스 배치 자동화 아키텍처 직접 적용 — 브리핑봇/이상치봇/매핑봇을 각 pane에 분리, 오케스트레이터가 조율하는 구조로 병목 해소 및 실패 격리.
