## 🧑‍💻 남들은 이렇게 쓴다 (활용 사례 Top 5)

- [thedotmack/claude-mem — Persistent Context Across Sessions](https://github.com/thedotmack/claude-mem) · GitHub
    - 사례: Claude Code 세션 종료 후에도 컨텍스트 소실 없이 이어가는 메모리 플러그인. 세션 중 tool 사용/결과를 자동 캡처·압축하고, 다음 세션 시작 시 관련 과거 작업을 RAG로 주입. 첫 조회는 ~50토큰 요약, 필요 시 ~500토큰 상세 로딩으로 10x 토큰 절감.
    - 접목: 헤드리스 이상치 탐지 배치에서 "지난주 어떤 anomaly 패턴이 있었나" 재주입 자동화. 매핑봇이 이전 매핑 결정을 기억해 반복 설명 제거. multi-agent 워크플로우에서 워커 세션 간 누적 지식 전달.

- [suisya-systems/claude-org-ja — 일본어 퍼스트 멀티에이전트 운영 프레임워크](https://github.com/suisya-systems/claude-org-ja) · GitHub
    - 사례: Claude Code 하나로 다수 Worker 인스턴스에 작업 위임. Secretary(사용자 창구·태스크 분해) → Dispatcher(Worker 초기화·배정) → Worker(격리된 디렉토리에서 실제 작업) → Curator(누적 학습 정리) 4역할 경계 명확화. 인간은 Secretary에게만 자연어로 지시.
    - 접목: Slack 브리핑 시스템 리팩토링에 직접 적용. Secretary가 "오늘 게임 KPI 요약", Dispatcher가 DAU/매출/온체인 쿼리를 각 Worker에 병렬 배정, Curator가 반복 분석 패턴을 스킬로 누적.

- [shanraisshan/claude-code-best-practice — vibe coding에서 agentic engineering으로](https://github.com/shanraisshan/claude-code-best-practice) · GitHub
    - 사례: Claude Code 실전 운영 가이드. Context 40% 이하 유지, `/compact` + 힌트 사용(자동 컴팩션 대신), 서브에이전트로 tool output 격리(부모 세션 누적 방지), `CLAUDE.md` 200줄 이내 + `.claude/rules/` YAML 조건부 로딩, 실패 시 `/rewind`로 롤백.
    - 접목: 배치 자동화에서 context rot 방지 핵심. 이상치 탐지 에이전트가 오래 돌수록 품질 저하하는 문제를 context 40% 룰 + 서브에이전트 격리로 해결. CLAUDE.md에 Snowflake 스키마·분석 관례 압축 문서화.

- [88plug/scientific-method — 반박 우선 가설 검증 워크플로우](https://github.com/88plug/scientific-method) · GitHub
    - 사례: Claude Code에 과학적 방법론 주입. 가설 → 예측 → 실험 → REFUTE(반박 공격) → 평결 → 레저 6단계. `EXPERIMENTS.md`에 모든 판정 영속화해 "한번 기각된 아이디어 재검토" 방지. `/scientific-method:investigate <문제>` 명령으로 실행.
    - 접목: 매크로 탐지 알고리즘 신규 개발 시 적용. "이 패턴이 정말 매크로인가"를 REFUTE 단계에서 adversarial하게 공격한 후에만 레이블링. 이상치 임계값 튜닝에서 과거 실험 레저로 중복 탐색 방지.

- [winchzyz/idea-to-build — 멀티에이전트 브레인스토밍 + 빌드 핸드오프](https://github.com/winchxyz/idea-to-build) · GitHub
    - 사례: 아이디어 → 실행 계획 6단계 자동화. 핵심은 Critique 단계 격리: 비평 에이전트가 forced premortem(실패 모드 열거), steelman(최강 반론), inversion(가정 뒤집기)를 독립 수행해 동의 편향 차단. `/scaffold`로 Claude Code 실행 디렉토리 자동 생성.
    - 접목: 다이나믹 프라이싱 신규 모델 설계 시 활용. "이 가격 모델이 왜 실패하는가"를 먼저 공격한 후 구현 착수. 분석 기능 spec 대신 빠른 프로토타입 20-30개 반복 권장 → 게임 데이터 분석 PoC에 적합.
