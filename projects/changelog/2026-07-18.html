🔄 Claude Code 신규 패치

v2.1.214 released 2026-07-18

보안 · 권한
  • Edit(src/**) 같은 단일 세그먼트 dir/** 허용 규칙이 <cwd>/dir 하위가 아닌 임의 경로에도 적용되던 권한 우회 버그 수정
  • Windows PowerShell 5.1 세션에서 권한 검사 우회 가능하던 버그 수정
  • 파일 디스크립터 리다이렉트, 10,000자 초과 명령, zsh 변수 서브스크립트 등 Bash 권한 분석기가 잘못 허용하던 다수 엣지 케이스 수정
  • docker / Podman 데몬 리다이렉트 플래그(--url, --connection 등)를 포함한 명령에 권한 프롬프트 추가
신규 기능
  • EndConversation 도구 추가 — 극도로 욕설적이거나 탈옥을 시도하는 사용자와의 세션을 Claude가 직접 종료 가능 (claude.ai 기존 동작과 통일)
  • 장시간 실행 도구 호출에 대한 주기적 진행 상황 하트비트 출력 추가 — 이전에는 무응답 상태로 침묵
  • 메모리 파일 프론트매터에 ISO 형식 modified 타임스탬프 추가
  • OpenTelemetry 로그 이벤트에 message.uuid, client_request_id, tool_source 속성 추가로 메시지 수준 추적 강화
  • CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH 환경변수로 OTel 콘텐츠 속성 60 KB 잘림 한계 조정 가능
  • subagentStatusLine 페이로드에 추론 노력(reasoning effort) 포함 — 커스텀 에이전트 행에서 모델·노력 수준 표시 지원
버그 픽스 (안정성)
  • GrowthBook 플래그가 null이면 크래시 / 잘못된 페이로드로 캐시 전체 초기화되던 문제 수정
  • pkill -f 패턴이 Claude CLI 자신의 프로세스를 종료하던 버그 수정 (Linux)
  • --settings가 장치 파일이나 2 MiB 초과 파일을 가리킬 때 무한 메모리 증가 방지 — 시작 시 명확한 오류 출력
  • Windows 기업 프록시 환경에서 스트리밍 "Socket is closed" 실패 수정
  • 느린 SDK·파이프라인 소비자를 위한 스트림 JSON 출력 잘림 수정 — 종료 드레인을 대기 바이트 기반으로 조정
  • 예약 작업이 자체 설정 프롬프트를 신뢰하지 않아 실행 거부하던 버그 수정
  • PowerShell 도구 자식 프로세스 표준 입력 대기로 타임아웃까지 행 걸리던 문제, UnicodeDecodeError/UnicodeEncodeError 크래시, >/>>가 UTF-16LE 파일 생성하던 문제 수정 (Windows)
  • 메모리 파일 저장 시 인라인 # 이후 프론트매터 값이 잘리던 버그 수정
  • 세션 비용·토큰 텔레메트리가 여러 message_delta 프레임에서 이중 계산되던 문제 수정
  • MCP 서버 슬래시 명령·리소스가 일시적 오류로 초기화되던 버그 수정
백그라운드 세션 · 기타
  • 배경 데몬이 후계 데몬의 제어 소켓을 삭제하던 경쟁 조건 수정
  • ← / /background로 주차된 세션이 데몬·워커 프로세스를 무한 유지하던 메모리 누수 수정
  • 비-git 폴더에서 실행된 백그라운드 세션을 에이전트 뷰에서 삭제 불가능하던 문제 수정
  • 플러그인이 --settings CLI 플래그로 활성화한 경우 로딩되지 않던 회귀 수정 (v2.1.181 이후)
  • OAuth 토큰 갱신 후 장시간 세션에서 기능 플래그 오래된 값 유지 문제 수정
  • SessionStart 훅에서 세션이 포크로 시작될 때 소스를 "fork"로 올바르게 보고하도록 변경
