🔄 Claude Code 신규 패치

v2.1.212 released 2026-07-17

신기능

  • /fork가 이제 대화를 새 백그라운드 세션으로 복사하고 기존 작업을 계속 진행 가능; 이전의 인세션 서브에이전트는 /subtask로 분리
  • claude auto-mode reset 명령 추가 — 기본 auto-mode 설정을 확인 프롬프트와 함께 복원 (--yes로 생략 가능)
  • 세션당 WebSearch 호출 횟수 제한 추가 (기본 200회, CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION으로 조정) — 무한 검색 루프 방지
  • 세션당 서브에이전트 스폰 횟수 제한 추가 (기본 200회, CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION으로 조정, /clear로 초기화) — 무한 위임 루프 방지
  • 2분 이상 실행되는 MCP 도구 호출이 자동으로 백그라운드로 이동 (CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS으로 임계값 조정 가능)
  • 에이전트 뷰에서 /resume 입력 시 삭제된 과거 세션을 포함한 선택 기능 제공 — 선택한 세션을 백그라운드 세션으로 재개

버그픽스

  • 플랜 모드에서 touch, rm 등 파일 수정 Bash 명령이 권한 프롬프트 또는 SDK canUseTool 콜백 없이 자동 실행되던 문제 수정
  • worktree 생성 시 .claude/worktrees 심볼릭 링크를 따라 저장소 외부에 파일이 생성될 수 있던 보안 문제 수정
  • 대량 이미지 포함 대화에서 "Request too large" 오류가 잘못 표시되던 문제 및 에러 메시지 개선
  • 웹 검색/페치가 API 과부하 시 에러 텍스트를 검색 결과나 페이지 내용으로 반환하던 문제 수정
  • 스트리밍 중 도구 실패/완료 시 continue:false 훅의 중단 신호가 무시되던 문제 수정
  • SIGTERM 수신 시 Bash 도구의 프로세스 트리가 고아 상태로 남던 문제 수정 — 이제 클린 종료 후 143 코드로 종료
  • 오픈텔레메트리 HTTP 익스포트가 Azure Monitor 등 청크 전송 인코딩을 지원하지 않는 엔드포인트에서 411/400 오류를 반환하던 문제 수정

성능 개선

  • 웹 검색과 웹 페치 신뢰성 향상 — 529 에러 및 rate-limit 요청에 대해 제한적 백오프로 재시도
  • 프롬프트 캐싱 개선 — LLM 게이트웨이 및 커스텀 base URL(Bedrock, Vertex, 1P)에서 mid-conversation 시스템 블록 정상 작동
  • SendMessage 본문이 재생 히스토리에 중복되지 않도록 개선하여 inter-agent 메시징 토큰 사용량 감소
  • 백그라운드 에이전트 콜드 어태치 시 세션 부팅을 기다리는 빈 화면 대신 즉시 트랜스크립트를 표시
